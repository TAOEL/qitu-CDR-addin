| PC 插件 | Web 后端 | HTTPS + SSE | AI 流式响应（Server-Sent Events） |

| 优势 | 说明 |
|------|------|
| 快速搭建 | 内置用户系统、权限管理、REST API 基础设施 |
| 插件生态 | 成熟的自定义插件机制，扩展方便 |

## 四、核心模块设计

### 4.1 PC 插件端架构（MVVM）

```
View (XAML)  ←──数据绑定──→  ViewModel (C#)  ←──调用──→  Service (C#)
     │                            │                        │
     │  用户交互                   │  业务逻辑编排           │  HTTP / CDR API
     │                            │                        │
     ▼                            ▼                        ▼
  UI 渲染                     状态管理                 外部通信
```

**关键约束**：

- CDR 版本适配：使用策略模式 + 版本适配器处理多版本 API 差异
- CDR 版本规则：**开发与测试仅针对 V26（CDR 2024），禁止使用 V27（CDR 2025）**，其他版本仅做兼容性验证
- 插件线程模型：所有 CDR 对象操作必须在主线程执行，耗时操作（网络/AI）必须在后台线程

### 4.2 Web 后端 API 设计

| 端点前缀 | 功能域 | 认证方式 |
|----------|--------|----------|
| `/wp-json/qitucdr/v1/auth/*` | 注册/登录/Token 管理 | 无（登录接口）/ Bearer Token |
| `/wp-json/qitucdr/v1/user/*` | 用户信息/配置 | Bearer Token |
| `/wp-json/qitucdr/v1/subscription/*` | 订阅套餐/状态 | Bearer Token |
| `/wp-json/qitucdr/v1/payment/*` | 创建订单/查询/回调 | Bearer Token / 网关签名 |
| `/wp-json/qitucdr/v1/sync/*` | 配置上传/下载 | Bearer Token |
| `/wp-json/qitucdr/v1/llm/*` | AI 文案/排版/色彩 | Bearer Token |

### 4.3 数据流

#### 用户认证数据流

```
插件输入账号密码
  → AuthService.Login()
  → ApiClient.Post(/auth/login, {email, password})
  → WordPress 验证 → 生成 JWT Token
  → 返回 Token + 用户信息
  → TokenManager.Store(token)  // 加密存储到本地
  → ViewModel 更新登录状态
```

#### AI 调用数据流

```
用户在 CDR 选中文字 → 点击"文案改写"
  → Plugin 收集 {text, context, action}
  → ApiClient.Post(/llm/rewrite, payload, stream=true)
  → WP LLM Handler 转发到 OpenAI API
  → 流式返回结果给插件
  → 插件实时展示并应用到 CDR 文档
```

### 4.4 数据库核心表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| wp_users | WordPress 内置用户表 | ID, user_email, user_pass |
| wp_qitucdr_subscriptions | 订阅记录 | user_id, plan, status, start_date, end_date |
| wp_qitucdr_orders | 订单记录 | user_id, amount, payment_method, status, created_at |
| wp_qitucdr_sync_data | 配置同步 | user_id, config_key, config_value, updated_at |
| wp_qitucdr_versions | 版本管理 | version, release_date, changelog, download_url |
| wp_qitucdr_llm_logs | AI 调用日志 | user_id, model, prompt_tokens, cost, created_at |

## 五、安全设计

| 层级 | 措施 |
|------|------|
| 传输层 | HTTPS 全站加密 |
| 认证层 | JWT Token，过期时间 7 天，支持 Refresh Token |
| API 层 | Rate Limiting（每分钟 60 次/用户），参数校验 + SQL 注入防护 |
| 支付层 | 支付数据不落业务库，仅存订单号和状态 |
| 插件端 | Token 本地加密存储（DPAPI），离线模式数据隔离 |
| AI 层 | 每日调用次数/费用预算上限，超限自动熔断 |
| 运维层 | 数据库每日自动备份，保留最近 7 天 |

           │

     └────────────┘
