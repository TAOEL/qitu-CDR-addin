## 一、产品概述

### 1.1 产品定位

QiTuCDR 是一套面向平面设计师、广告制作人员、包装结构师的 **CorelDRAW 增强工具箱插件**，辅以官网、用户中心及后台管理系统，形成完整的工具+服务生态。

| 维度 | 说明 |
|------|------|
| 产品名称 | 企图插件（QiTuCDR） |
| 产品类型 | CorelDRAW 插件 + Web 服务系统 |
| 目标用户 | 平面设计师、广告制作人员、包装结构师 |
| 核心价值 | 50+ 即用工具，AI 智能辅助，云端配置同步 |
| 商业模式 | 免费基础功能 + 付费订阅解锁完整版 |
| 版本号 | V1.0 |
| 开发周期 | 16 周（4 个月） |

### 1.2 产品边界

| 范围 | 内容 |
|------|------|
| 包含 | PC 端 CDR 插件、Web 官网与用户中心、管理后台、AI 辅助模块 |
| 不包含 | 移动端 App、第三方设计软件插件（AI/PSD 等） |
| V1.0 包含 | 50+ 工具功能、用户注册/登录/订阅/支付、AI 文案改写/排版建议/色彩推荐 |
| V1.0 不包含 | 协作功能、插件市场、开放 API |

## 四、用户流程

### 4.1 核心用户旅程：从发现到付费使用

```
访客浏览官网 → 下载插件 → 安装插件 → 体验免费功能
    → 注册账号 → 选择套餐 → 完成支付 → 解锁全部功能
    → 使用 AI 辅助 → 配置云同步 → 持续使用/续费
```

### 4.2 PC 端插件启动流程

```
启动 CDR → 加载 QiTuCDR 插件 → 检查登录状态
    ├─ 已登录 → 检查订阅有效期
    │   ├─ 有效 → 加载完整功能 + 同步云端配置
    │   └─ 已过期 → 降级为免费模式 + 提示续费
    └─ 未登录 → 加载免费功能 + 显示登录入口
```

### 4.3 登录注册流程

```
插件登录页 → 输入邮箱 + 密码 → 调用 API 验证
    → 获取 Token → 存储本地 → 刷新 UI 状态
    └─ 无账号 → 切换注册页 → 输入邮箱 → 获取验证码
        → 设置密码 → 注册成功 → 自动登录
```

### 4.4 支付订阅流程

```
插件/Web 端选择套餐 → 点击订阅
    → 创建订单（后端）
    → PC 端：生成支付二维码，用户扫码支付
    → Web 端：跳转支付网关
    → 支付成功回调 → 更新用户订阅状态
    → 插件端刷新 → 解锁付费功能
```

### 4.5 AI 功能调用流程

```
用户在 CDR 中选中文字/对象 → 点击 AI 功能按钮
    → 插件收集上下文（选中内容 + 文档信息）
    → 发送到后端 API → 后端转发到大模型
    → 大模型返回结果 → 后端处理后返回插件
    → 插件应用结果到 CDR 文档
```

## 六、MVP 迭代开发计划

### 6.1 迭代原则

| 原则 | 说明 |
|------|------|
| PC 先于 Web | 所有 P0/P1 功能先在 PC 端完成并验证，再扩展到 Web 端 |
| UI 先于交互 | 每个功能模块先搭建可展示的静态 UI，再接入后端交互逻辑 |
| 单切片可验证 | 每个迭代产出是一个独立可测试、可演示的最小功能单元 |
| 测试驱动前进 | 当前切片测试跑通并验收后，才能进入下一个切片 |
| 文档同步 | 功能或架构有重大变更时，同步更新 PRD / 架构 / 项目状态三份文档 |

## 七、非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 插件启动时间 < 2 秒，工具操作响应 < 500ms，AI 调用 < 5 秒 |
| 兼容性 | CDR X7 ~ 2026（12 个版本），Windows 10/11 x64 |
| 安全 | 密码加密存储、API Token 机制、HTTPS 传输、支付数据不落库 |
| 可用性 | 插件崩溃不影响 CDR 主程序，Web 端可用性 ≥ 99.5% |
| 可维护性 | 模块化架构、注释覆盖率 ≥ 60%、错误码统一管理 |
